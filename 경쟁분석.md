# 경쟁 분석: Realtime Flowchart for VS Code 익스텐션

## 1. 레퍼런스 및 유사 익스텐션 분석

1.  **Code Flow:** (VS Code Marketplace)
    *   **기능:** 주로 함수 호출 흐름(Call Graph)을 시각화하는 데 중점을 둬. 특정 함수가 어떤 함수를 호출하고, 또 어떤 함수에 의해 호출되는지를 그래프 형태로 보여줘.
    *   **우리 익스텐션과의 차이점:** 우리는 '함수 내부의 조건/분기 로직(if-else, switch)과 데이터 흐름'에 더 초점을 맞추고 있어. 즉, 미시적인 코드 블록 단위의 흐름 분석에 강점이 있지.

2.  **Code Outline / Structure View (VS Code Built-in):**
    *   **기능:** 코드의 전체적인 구조(함수, 클래스, 변수 등)를 트리 형태로 보여주지만, '흐름' 자체를 시각화하지는 않아.
    *   **우리 익스텐션과의 차이점:** 정적인 구조가 아닌 동적인 실행 흐름과 데이터의 움직임을 시각화하는 것이 우리의 목표야.

3.  **Debugger Visualizer (다양한 형태로 존재):**
    *   **기능:** 디버깅 시점에 변수의 값 변화나 스택 트레이스 등을 시각적으로 보여주는 도구들.
    *   **우리 익스텐션과의 차이점:** 우리는 '실시간 정적 분석'을 통해 코드를 작성하는 도중에 미리 흐름을 파악하는 것이 목표야. 디버깅 시점에만 동작하는 것이 아니지.

4.  **Flowchart for VS Code:**
    *   **기능:** 일부 언어의 특정 구문을 기반으로 정적인 순서도를 생성하는 익스텐션들이 있지만, '실시간' 및 '데이터 흐름 분석'까지는 미치지 못하는 경우가 많아.

## 2. 우리의 차별점

*   **실시간 + 제어 흐름 + 데이터 흐름 통합:** 코드를 작성하는 도중에 `if-else`/`switch` 같은 제어 흐름은 물론, 변수의 라이프사이클과 같은 데이터 흐름까지 동시에 시각화하여, 코드가 `꼬이는` 순간을 바로 포착할 수 있다는 점.
*   **직관적인 로직 파악:** 복잡한 중첩 로직을 코드를 읽는 노력 없이도 한눈에 파악할 수 있도록 돕는 것이 핵심이야.
